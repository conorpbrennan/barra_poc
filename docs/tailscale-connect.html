Barra Factor-Risk — Private Access

Connecting to the risk dashboard & analysis notebook over a private Tailscale network

These services run on a private machine and are not on the public internet. You reach them through Tailscale — a zero-config private network (WireGuard-based). It takes about three minutes to set up: install the app, sign in with the invite, and open the links. No VPN config, no firewall changes.

Before you start

You'll receive an email invitation to the network from Tailscale (sender noreply@tailscale.com). Keep it open — you'll click Accept in Step 2. If you don't see it, check spam, or ask your contact (below) to re-send.

1

Install Tailscale

Install the client for whichever device you'll use to view the dashboard.

Windowstailscale.com/download/windows
macOSApp Store / tailscale.com/download/mac
iPhone / iPadApp Store — search “Tailscale”
AndroidGoogle Play — search “Tailscale”
Linuxcurl -fsSL https://tailscale.com/install.sh | sh
2

Accept the invite & sign in

Open the invitation email and click Accept invitation. It opens Tailscale and asks you to sign in (Google, Microsoft, GitHub, or email — whichever the invite offers). Use the same identity the invite was sent to.

Then open the Tailscale app and make sure it shows Connected (the icon is solid, not greyed).

3

Open the apps

With Tailscale connected, open these links in your browser. They are served over HTTPS with a valid certificate — no security warnings, no passwords for the dashboard.

Troubleshooting

The link won't load / “can't reach this site”

Open the Tailscale app and confirm it says Connected. If it's off, toggle it on and try again. On a phone, make sure the Tailscale VPN profile is active (a small key/VPN icon in the status bar).

It asks me to sign in to Tailscale again

That's normal after a restart — just sign in with the same account you used to accept the invite.

The dashboard machine seems offline

The host machine may be asleep. Ping your contact to wake it; once it's up the link responds immediately.

I get a certificate warning

You shouldn't — these use real Tailscale-issued HTTPS certs. If you see one, you've likely typed http:// or an IP address; use the exact https://…ts.net links above.
